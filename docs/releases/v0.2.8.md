# Kida 0.2.8 Release Notes

**Release date:** TBD

## Highlights

### `{% yield %}` Directive for Slot Forwarding

New `{% yield %}` directive for explicit slot forwarding in nested macro composition. Unlike `{% slot %}`, which becomes a content definition (`SlotBlock`) inside `{% call %}` blocks, `{% yield %}` always produces a render reference (`Slot`) regardless of block context.

```kida
{% def selection_bar() %}<bar>{{ caller() }}</bar>{% end %}
{% def resource_index() %}
  {% call selection_bar() %}
    {% yield selection %}
  {% end %}
{% end %}
{% call resource_index() %}
  {% slot selection %}Badges{% end %}
  Cards
{% end %}
```

- `{% yield %}` ‚Äî render caller's default slot
- `{% yield name %}` ‚Äî render caller's named slot

This replaces the fragile double-nesting workaround `{% slot x %}{% slot x %}{% end %}` with clear, context-independent syntax.

### Render-Path Performance Optimizations

A series of optimizations targeting `auto_reload` overhead and hot-path allocation:

- **mtime-based stale checks** ‚Äî `auto_reload` now checks file mtime before reading and hashing source, skipping the slow path when the file hasn't been modified.
- **Pre-computed block wrappers** ‚Äî `CachedBlocksDict` builds wrapper functions once at init instead of creating a new closure on every `.get()` call.
- **Single-pass context merge** ‚Äî `_build_render_context` uses `{**globals, **args, **kwargs}` instead of filtering `UNDEFINED` values and calling `.update()` multiple times.
- **`lookup_scope` fast path** ‚Äî Skips the `reversed()` iterator when `scope_stack` is empty (the common case at top level).
- **Single-pass tag counting** ‚Äî `estimate_template_weight` scans the source once instead of four separate `str.count()` calls.

### Region Block Dispatch and Default Expressions

- **Region `_blocks` parameter** ‚Äî Region callables now receive the block dispatch dict, enabling regions to render inherited blocks.
- **Region default expressions** ‚Äî Optional region parameters support arbitrary expressions as defaults (e.g. `meta=page.metadata`, `count=items | length`). Defaults are evaluated at call time from the caller's context.

### Imported Macro Namespace Injection

Imported macros now carry their defining template's namespace, so cross-template macro calls (e.g. `article_card` calling `tag_list` from the same import) resolve correctly without requiring the caller to import every dependency.

## What's New

- **`{% yield %}` directive** ‚Äî Context-independent slot rendering for nested calls
- **Region default expressions** ‚Äî Arbitrary expressions as region parameter defaults
- **Region `_blocks` param** ‚Äî Regions receive block dispatch for inherited block rendering
- **Imported macro namespaces** ‚Äî Macros see sibling macros from their defining template
- **Render-path performance** ‚Äî mtime stale checks, pre-computed block wrappers, single-pass context merge, scope lookup fast path, single-pass tag counting
- **Regions example** ‚Äî New `examples/regions/` with app, templates, and tests
- **Docs** ‚Äî Functions reference updated with yield syntax and semantics

## Upgrade Notes

No breaking changes. All additions are backward-compatible. Existing templates continue to work unchanged.
