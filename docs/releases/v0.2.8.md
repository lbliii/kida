# Kida 0.2.8 Release Notes

**Release date:** TBD

## Highlights

### `{% yield %}` Directive for Slot Forwarding

New `{% yield %}` directive for explicit slot forwarding in nested macro composition. Unlike `{% slot %}`, which becomes a content definition (`SlotBlock`) inside `{% call %}` blocks, `{% yield %}` always produces a render reference (`Slot`) regardless of block context.

```kida
{% def selection_bar() %}<bar>{{ caller() }}</bar>{% end %}
{% def resource_index() %}
  {% call selection_bar() %}
    {% yield selection %}
  {% end %}
{% end %}
{% call resource_index() %}
  {% slot selection %}Badges{% end %}
  Cards
{% end %}
```

- `{% yield %}` — render caller's default slot
- `{% yield name %}` — render caller's named slot

This replaces the fragile double-nesting workaround `{% slot x %}{% slot x %}{% end %}` with clear, context-independent syntax.

## What's New

- **`{% yield %}` directive** — Context-independent slot rendering for nested calls

## Upgrade Notes

No breaking changes. `{% yield %}` is additive. Existing templates continue to work unchanged.
