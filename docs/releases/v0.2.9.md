# Kida 0.2.9 Release Notes

**Release date:** 2026-03-23

## Highlights

### `kida check` CLI

New `kida check <template_dir>` command parses every `*.html` file under a directory using the
filesystem loader, catching syntax errors and resolution failures before runtime.

- **`--strict`** — Re-parses sources and warns on unified `{% end %}` closers, suggesting explicit
  tags such as `{% endblock %}` or `{% enddef %}`.
- **`--validate-calls`** — When templates compile successfully, runs `BlockAnalyzer` to compare
  macro call sites against `{% def %}` signatures (unknown parameters, missing required arguments,
  duplicate keyword arguments).

### `render_block()` and ancestor imports

Block-only rendering now resolves macro imports from ancestor templates the same way as full
template loads, so inherited `{% from %}` patterns work inside `render_block()` flows.

### Compiler and benchmarks

- Fewer redundant AST traversals during f-string coalescing and related compiler passes.
- `benchmarks/RESULTS.md` updated for Python 3.14 free-threading measurements.
