# Kida 0.3.2 Release Notes

**Release date:** 2026-04-03

## Highlights

### Compiler Intelligence

The partial evaluator now handles significantly more node types — `Def`, `FuncCall`, `MarkSafe`,
`NullCoalesce`, `OptionalFilter`, `SafePipeline`, `Slot`, and `SlotBlock` — enabling deeper
compile-time optimization. Templates with static context see more expressions resolved to constants
before codegen.

### Python 3.14 Modernization

`@final` decorators on key classes prevent accidental subclassing in free-threaded code. `match`
statements replace `if`/`elif` chains in the partial evaluator and constant evaluator for clearer
dispatch. Stricter ruff linting catches more issues at development time.

### Quality & Performance

Consolidated AST traversals in dependency analysis (single-pass instead of multiple walks), cached
AST in the bytecode cache for faster recompilation, and deduplicated render helpers reduce overhead
across the compile-render pipeline.

### New CLI Commands

- `kida explain <template>` — Shows which compile-time optimizations are active for a template.
- `kida bench <template>` — Quick compile and render performance measurement.
