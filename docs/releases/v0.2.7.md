# Kida 0.2.7 Release Notes

**Release date:** 2026-03-12

## Highlights

### Fail-Fast Filter Errors and Better Suggestions

Kida 0.2.7 tightens several filter failure modes so template mistakes surface as actionable
runtime errors instead of quietly stringifying bad input. `join` and `reverse` now require
iterable input, `pluralize` rejects non-numeric values, `replace` coerces string counts from
YAML/config, and `decimal` / `format_number` add `strict=True` for explicit numeric conversion
failures.

Runtime error enhancement also got smarter: Kida now adds targeted suggestions for macro/context
name collisions, `%` vs `{}` format mistakes, invalid numeric coercion, `strftime` formatting,
and sequence-only filters.

### Nested Slot Passthrough Fix

Nested component chains now preserve outer slot content when an inner slot body is empty or
whitespace-only. This fixes layout patterns where wrapper components provide pass-through slots to
deeper macros or defs.

```kida
{% call resource_index("Skills", "/skills") %}
  {% slot filter_controls %}<button>Filters</button>{% end %}
  {% slot filter_actions %}<a href="#">Clear</a>{% end %}
  <article>Skill A</article>
{% end %}
```

If an intermediate macro leaves `filter_controls` or `filter_actions` empty, Kida now correctly
delegates to the outer caller instead of treating the empty slot as final content.

### `typeof` Filter for Debugging

New `typeof` support helps inspect mixed data flowing into templates:

```kida
{{ value | typeof }}
```

It returns normalized names like `bool`, `int`, `float`, `path`, `list`, `dict`, `none`, and
`str`.

## What's New

- **`typeof` filter** — Stable type names for common template values and config data
- **Stricter filter contracts** — `join` and `reverse` now fail fast on non-iterables
- **Numeric coercion improvements** — `replace` count coercion plus `decimal(..., strict=true)`
  and `format_number(..., strict=true)`
- **Macro collision diagnostics** — Better errors when a macro shadows a context variable and is
  used as an iterable
- **Nested slot passthrough fix** — Empty inner slots now delegate to the outer caller correctly
- **Docs updates** — Release notes, changelog, and filter reference refreshed for the new behavior

## Upgrade Notes

1. **Review `join` and `reverse` usage** — Templates that relied on implicit string fallback now
   need iterable input.
2. **Use `| int` / `| float` for config values when needed** — This is especially helpful for
   YAML-driven counts and numeric formatting.
3. **Rename macros that shadow context variables** — Prefer verb-prefixed names like
   `render_route_tabs`.
