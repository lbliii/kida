# Kida 0.3.0 Release Notes

**Release date:** 2026-03-26

## Highlights

### Terminal Rendering Mode

`Environment(autoescape="terminal")` enables declarative terminal/CLI output. Includes ANSI-safe
escaping, 30+ terminal filters (colors, badges, tables, trees, diffs), icon/box-drawing character
sets with Unicode/ASCII degradation, and reusable built-in components (panel, header, footer, rule,
connector, row, cols).

S-tier enhancements include configurable `WidthStrategy` with terminal probe auto-detection, color
depth fallback (`fg()`/`bg()` degrade across truecolor → 256 → basic → none), `LiveRenderer` for
in-place re-rendering with `Spinner` animation, `stream_to_terminal()` for progressive output, and
responsive `stack()` layout.

### Framework Integrations

Drop-in adapters in `kida.contrib` for Flask, Starlette/FastAPI, and Django.

### Sandboxed Environment

`SandboxedEnvironment` restricts collection methods to read-only, blocks calls to unsafe types, and
provides `allow_mutating_methods` and `allow_calling` allowlists for configurable security policies.

### Template Tooling

- **Coverage collector** — `CoverageCollector` tracks template line execution with LCOV/Cobertura
  export. Thread-safe for concurrent usage.
- **Accessibility linting** — Static checks for `img-alt`, `heading-order`, `html-lang`,
  `input-label`.
- **Type checker** — `{% template %}` declarations for context type annotations.
- **Formatter** — `kida fmt` for opinionated template formatting.
- **Extensions** — Plugin architecture with `Extension` base class, `node_types` registration,
  and `_extension_compilers` dispatch.

### Engine Improvements

- Immutable frozen-dataclass AST with structural equality.
- Free-threading safe `RenderContext` (no shared mutable state).
- Compiler optimizations: constant folding, output coalescing, copy-on-write AST transforms,
  hoisted dispatch tables, compile-time rebinding tracking.
- Content stacks/portals (`{% push %}` / `{% stack %}`).
- CSP nonce support for inline scripts/styles.
- `kida render` CLI command.
