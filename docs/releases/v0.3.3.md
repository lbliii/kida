# Kida 0.3.3 Release Notes

**Release date:** 2026-04-07

## Highlights

### syntax() Terminal Filter

New `syntax()` filter for terminal rendering mode provides regex-based syntax highlighting
for JSON and YAML content. Keys, strings, numbers, and structural characters are
color-coded using ANSI escape sequences.

### PR Comment CI Reports

The GitHub Action now supports posting CI reports as pull request comments via
`post-to: pr-comment`. Comments are deduplicated using `comment-header` so re-runs update
in place instead of creating duplicates. Use `comment-mode: append` to combine multiple
reports (e.g., pytest + coverage) into a single comment. The new `context` input lets you
pass extra template variables as a JSON string.

### CachedBlocksDict Fix

Fixed a bug where `CachedBlocksDict` did not properly track cache misses during block
lookups. Added comprehensive test coverage for cached blocks, purity checks, and t-strings.

## What's Changed

### Added

- `syntax()` terminal filter with JSON and YAML highlighting
- PR comment posting in GitHub Action (`post-to: pr-comment`)
- Comment deduplication via `comment-header`
- Append mode (`comment-mode: append`) for combining reports
- Extra context injection via `context` JSON input
- Expanded test suites for cached blocks, purity, and t-strings

### Fixed

- `CachedBlocksDict` miss tracking — cache misses now properly recorded

## Upgrade Notes

No breaking changes. Drop-in upgrade from 0.3.2.
