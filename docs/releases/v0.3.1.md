# Kida 0.3.1 Release Notes

**Release date:** 2026-04-02

## Highlights

### Null-Safe Operators

Four ergonomic operators for safer, more concise template authoring:

- **Safe pipeline (`?|>`)** — None-propagating filter chain. If the value is None, all subsequent
  filters are skipped: `{{ user?.name ?|> upper ?|> trim ?? "Anonymous" }}`.
- **Optional filter (`?|`)** — Skip a single filter when the value is None:
  `{{ value ?| upper ?? "N/A" }}`. Preserves falsy values like `0` and `""`.
- **Nullish assignment (`??=`)** — Assign only if the variable is undefined or None:
  `{% let title ??= "Untitled" %}`.
- **`promote` keyword** — Alias for `export` that reads more naturally for scope promotion.

### Markdown Rendering

`autoescape="markdown"` provides markdown-safe escaping and CI report filters for rendering
structured data as Markdown documents.

### GitHub Action

`lbliii/kida@v0.3.1` renders CI reports as GitHub step summaries or PR comments. Built-in templates
for pytest, coverage, ruff, ty, jest, gotest, and SARIF output.

### CI Report Parsers

New parsers for SARIF, LCOV, and JUnit XML formats enable structured ingestion of CI tool output
for template rendering.
