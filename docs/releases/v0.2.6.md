# Kida 0.2.6 Release Notes

**Release date:** 2026-03-09

## Highlights

### `{% region %}` — Parameterized Blocks for Fragments and OOB

Regions are the headline feature in 0.2.6. They are parameterized blocks that work as **both** blocks (for `render_block()`) and callables (for `{{ name(args) }}`). Use them when you need parameterized fragments for HTMX partials, OOB updates, or layout composition.

```kida
{% region sidebar(current_path="/") %}
  <nav>{{ current_path }}</nav>
{% end %}

{% block content %}
  {{ sidebar(current_path="/about") }}
{% endblock %}
```

```python
# From Python: render the region as a block
html = template.render_block("sidebar", current_path="/settings")
```

Regions support required and optional params, default values, and access to outer render context. Frameworks like [Chirp](https://github.com/lbliii/chirp) use `template_metadata().regions()` to discover OOB regions at build time for AST-driven layout contracts.

### render_block + Def Scope Fix

Blocks can now call defs from the same template. Previously, `render_block("content")` would fail with `NameError` if the block used `{{ helper() }}` and `helper` was a `{% def %}` in the same file. Top-level defs and regions are now compiled into the globals setup, so both full-page `render()` and `render_block()` have access to them.

**Workaround no longer needed:** You no longer need to split defs into a separate file and import them just to use them from blocks. Use regions when you need parameterized blocks that use defs.
