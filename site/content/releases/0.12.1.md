# v0.12.1

**Released** 2026-07-17.

Kida 0.12.1 is a security release for applications using
`SandboxedEnvironment`. Applications on 0.12.0 should upgrade promptly.

## Security fixes

- **Callable policy enforcement** — Ordinary template function calls now pass
  through the sandbox callable policy. Arbitrary Python functions supplied
  only through render context are blocked, while application-registered
  globals and same-environment Kida component functions retain their intended
  trust. Explicit `allow_calling` allowlists still apply, including the empty
  allowlist that blocks every call.
- **Cumulative streamed output limits** — `max_output_size` now applies to
  `render_stream()`, `render_stream_async()`, and
  `render_block_stream_async()` as well as full and fragment rendering.

The sandbox remains defense-in-depth, not process isolation. Continue to
curate render context and apply process/container limits for adversarial
template source.

## Proof

- Focused `src/kida/sandbox.py` coverage is 100% for statements and branches.
- The full local stability gate passed on CPython 3.14t with the GIL disabled,
  including repository coverage, concurrency/safety suites, and installed
  wheel smoke tests.
- Ordinary `Environment` retains direct-call code generation and unrestricted
  stream performance remains within local benchmark variance.

See the published GitHub security advisory for impact, mitigations, and the
full technical analysis.
